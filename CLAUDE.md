# CLAUDE.md

Claude Code 用のプロジェクト文脈・規約。実装の前に必ず参照すること。

## このプロジェクトは何か

立体4目並べ（**Score Four / 重力あり 4×4×4**）の **対戦・解析エンジン**と**定石**を作る。
最優先ゴールは **最小労力で最強の解析エンジン**で、手法は **α-β 探索中心**。
方針の全体像は `docs/design.md`、ゲーム仕様は `docs/rules.md` にある。**着手前に両方読むこと。**

## 絶対に外してはいけない前提

1. **対象は重力あり版（Score Four）。Qubic（自由配置）ではない。**
   コマは選んだ柱の最下段に落ちる。任意の高さには置けない。着手＝「柱の選択」のみ。
2. **コアの正しさが全ての土台。**
   着手生成・勝利判定のバグは探索・学習・定石を静かに全部汚染する。
   → **総当たり参照実装との契約テストでコアを固めてから探索へ進む（センサー先行）。**
3. **勝利ラインは全 76 本**（軸 48・平面の斜め 24・立体対角線 4）。
   立体対角線（4 本）と段をまたぐ斜めの実装漏れに注意。

## 実装の順序（この順で進める）

1. `lines.py`：76 ラインを生成。**まず総当たりで全列挙し、本数が 76 であることをテストで保証。**
2. `board.py`：ビットボード（各色 64bit）、着手生成（空き柱）、勝利判定（直前に置いたセルを通る線のみの増分判定）。
3. `tests/`：ナイーブな参照実装（3 次元配列で愚直に判定）と、ランダム対局で**全局面一致**を property test で検証。
4. `search.py`：negamax + α-β → 置換表（Zobrist）→ 脅威枝刈り → 着手順序 → 対称性圧縮（D4・8 重）→ 反復深化 + PVS。
5. `evaluate.py`：脅威カウント ＋ 奇偶パリティ。**探索が終端に届く局面では使わない**ため後回し。
6. 定石生成：根から深い反復深化、PV と評価値を保存。

## コーディング規約

- Python 3.11+。まず Python で正しく作り、速度が律速なら**ホットループのみ** Rust/C++ へ移植する（早すぎる最適化はしない）。
- 盤面表現・セルのインデックス規約は `board.py` の冒頭 docstring に**唯一の定義**として書き、他はそれを参照する。
- 公開関数には型注釈と docstring。副作用のある関数と純粋関数を混ぜない。
- 乱数・探索の非決定性はテストで排除（シード固定、同一局面で同一手）。

## テストの方針（最重要）

- **新しいコア機能は、参照実装との一致テストを書いてからマージする。**
- 既知局面（即詰み・ダブルリーチ・引分確定）の回帰テストを増やしていく。
- `pytest` を緑にしてから次のレイヤへ進む。

## やらないこと

- 最初から AlphaZero に手を出さない（α-β が頭打ちになるまで不要）。
- 評価関数の作り込みを探索より先に進めない。
- コアのテストが緑になる前に探索を書き始めない。
