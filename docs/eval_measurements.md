# 評価関数の計測メモ

> docs/design.md 4.7 / CLAUDE.md の方針「評価ヒューリスティックは確立理論ではなく
> 仮説。**自己対戦で効果を測ってから採否・重みを決める**」に従う計測記録。
> ここに測定の手続きと結果、そこから下した判断を残す。

## 計測ハーネス

`src/score_four/selfplay.py`

- `random_openings(count, plies, seed)`: 相異なる非終端の序盤手順を生成。
- `play_match(eval_a, eval_b, openings, depth)`: 各 opening を**先後入れ替えて 2 局**
  打ち、先手有利を相殺して `(a_wins, b_wins, draws)` を返す。
- 探索は `time_limit` を使わず**固定 `max_depth`**、評価は決定的 → 同一条件は再現可能。
- winrate は B (= eval_b) 視点で `(b_wins + 0.5*draws) / total`。
- 統計の目安: 1 標準誤差 ≈ `sqrt(0.25 * total)` 局。winrate 0.5 から ±1SE/total
  以内の差は誤差レンジ。

## 仮説 1: 即時脅威 (immediate threat) への加点

`threat_eval(board, immediate)` は `line_potential` に「3 並びで残り 1 マスが今すぐ
着手できる」ラインへ `immediate` を加点する版 (`immediate=0` は `line_potential` と
完全一致)。基準 A = `line_potential`、B = `threat_eval(immediate=w)`。

### 結果

| depth | games | immediate=w | B winrate | 内訳 (B-A, draw) |
|------:|------:|------------:|----------:|------------------|
| 3 | 80  | 8  | 0.537 | 43-37, 0 |
| 3 | 80  | 16 | 0.450 | 36-44, 0 |
| 3 | 80  | 40 | 0.450 | 36-44, 0 |
| 3 | 240 | 4  | 0.515 | 123-116, 1 |
| 3 | 240 | 8  | 0.521 | 125-115, 0 |
| 3 | 240 | 12 | 0.533 | 128-112, 0 |
| 4 | 80  | 40 | 0.425 | 34-46, 0 |
| 4 | 180 | 10 | 0.475 | 85-94, 1 |

(1SE ≈ 4.5 局 @80、7.7 局 @240、6.7 局 @180)

### 所見

- **大きい immediate 重み (16〜40) はベースラインより明確に弱い** (winrate ≈ 0.42〜0.45)。
- **小さい重み (4〜12) は depth3 ではわずかに優勢** (winrate ≈ 0.51〜0.53) だが、
  effect は小さく 240 局でも 1〜2SE 程度の弱いシグナル。
- **decisive: depth4 では小さい重み (10) でも 0.475 と優位は消える** (むしろ僅かに不利)。
  depth3 の弱いプラスは**深さを上げると再現しない**。
- 解釈: 探索は `depth>=2` で即時の戦術 (即勝ち・ダブルリーチ) を**脅威枝刈りで正確に
  解決**する。深く読むほど探索自身が戦術を処理するので、地平線評価で即時脅威を
  足す価値は消え、重く足すとむしろ静的局面の判断を歪める。

## 判断

- **即時脅威の加点は頑健な改善にならない** → 採用しない。**既定の探索ヒューリスティックは
  引き続き `line_potential`**。
- `threat_eval` (即時脅威可変) / `parity_eval` は計測用・実験用に残す。
- 計測ハーネスが「もっともらしいが実は中立〜悪化する改変」を**採用前に検出した**好例。
  CLAUDE.md の「評価は探索より後・測ってから決める」が機能している。

## 仮説 2: 奇偶 (パリティ) — **検証され採用**

`parity_eval(board, parity_weight, immediate=0)` は各プレイヤーの未完成 3 並びの
「完成セルの高さ z」の偶奇 (奇数 +1 / 偶数 -1) を数え、先手 - 後手の差を
`parity_weight` 倍して先手視点スコアへ加える。パリティ効果を純粋に測るため
即時脅威の加点は切って (immediate=0) ベースライン `line_potential` 上で掃引した。
正の重み = 「奇数段の脅威が先手有利」(素朴な Connect Four 流)、負の重み = 逆向き。

### 結果 1: 重みの向きと大きさ (depth3, seed 313 で掃引)

| depth | games | seed | parity_weight | B winrate | 内訳 (B-A, draw) |
|------:|------:|-----:|--------------:|----------:|------------------|
| 3 | 180 | 313 | -20 | 0.392 | 69-108, 3 |
| 3 | 180 | 313 | **-10** | **0.542** | 96-81, 3 |
| 3 | 180 | 313 | -5  | 0.522 | 94-86, 0 |
| 3 | 180 | 313 | +5  | 0.478 | 86-94, 0 |
| 3 | 180 | 313 | +10 | 0.397 | 71-108, 1 |
| 3 | 180 | 313 | +20 | 0.361 | 65-115, 0 |

明確で単調な向き: **負の重みが勝ち、正の重みが負ける**。最適は概ね -5 〜 -10、
大きすぎる重み (|w|=20) は過剰で逆効果。

### 結果 2: シード分散と多シード集計 (w=-8 固定)

単一シードは振れる。fine sweep を別シード (2027) でやると全 -6〜-10 が 0.47〜0.50 と
**優位が消えた** ため、過学習を疑って多シード集計した。

| depth | 構成 | B winrate | 内訳 (B-A, draw) | 有意性 |
|------:|------|----------:|------------------|--------|
| 3 | seed 999, 180g | 0.533 | 96-84 | |
| 3 | seed 2027, 300g | 0.482 | 144-155 | (外れ値) |
| 3 | **seeds 101/202/303/404 集計 1200g** | **0.530** | 635-562 | ~2.1 SE |
| 4 | seed 313, 180g | 0.572 | 103-77 | ~4 SE |
| 4 | **seeds 505/606/707 集計 540g** | **0.593** | 318-218 | ~4.3 SE |
| 5 | seed 444, 110g | 0.505 | 55-54 | 標本過小 (±0.047) |

### 所見

- **頑健な改善 (集計で確定)**: depth3 で 1200 局集計 0.530 (~2.1SE)、depth4 で 540 局
  集計 0.593 (~4.3SE)。**いずれも複数の新規シードで全て勝ち越し**。単一シードの
  振れ (seed 2027 の 0.482) は外れ値で、集計すると明確に正。
- **深さで強まる**: depth3 (0.53) → depth4 (0.59)。即時脅威 (仮説1) が depth4 で
  優位消失したのと対照的。深く読むほどパリティの価値が出る。
- **depth5 は未確定**: 110 局では ±0.047 と粗く 0.505。否定の証拠ではない (0.53〜0.59 と
  矛盾しない) が、深部の精密確認は今後の課題 (計算コストが高い)。
- **教訓**: 単一シードの強い数字 (例: depth4 seed313 の 0.572) は鵜呑みにしない。
  多シード集計で初めて頑健性が言える。計測の途中、外れ値シードと粗い depth5 で
  一時「過学習では」と疑ったが、多シード集計が誤警報を解消した。
- **向きの意味**: 負の重み = 「偶数段 (z=0,2) の脅威が先手有利 / 奇数段 (z=1,3) が
  後手有利」。z は 0 始まりなので z=0 (最下段) は 1-index では「奇数段」。つまり
  「奇数段 (1-index) の脅威が先手有利」となり、**Connect Four のパリティ直観と整合**。

## 判断 (パリティ)

- **採用**。`_PARITY = -8` を既定とし、`default_eval = parity_eval(weight=-8, immediate=0)`
  を **探索の既定ヒューリスティック** にした (search/best_move のデフォルト)。
  多シード集計で depth3/4 とも有意 (~2〜4SE)、深さで強まる頑健な改善。
- D4 不変なので対称性圧縮 TT と矛盾しない (test_symmetry.py で検証)。
- 注意: これは「Score Four のパリティ理論」を確立した主張ではなく、固定深さ自己対戦
  での経験的勝率向上。

## 続報: 高速化(Rust)後の精緻化とトーナメント

探索の Rust 移植で約50〜60倍速くなり、depth5 で数千局を数分で回せるようになった。
評価をパラメータ化 (parity_weight / immediate / parity_mode) し、パリティ特徴を精緻化
して比較した。パリティモード (いずれも z 依存 = D4 不変):
  ALL=全 open-3 脅威 / REACHABLE=今すぐ着手可能な脅威のみ / LOWEST=各色の最下段脅威のみ。

### Stage A: 特徴×重みを vs baseline で掃引 (depth5, 300局/設定, 単一シード)

| feature | 最良 | 傾向 |
|---------|------|------|
| ALL | -4: 0.558, -6: 0.540 | -8: 0.503, -12: 0.458 (この単一シードでは小重みが上) |
| LOWEST | -8: 0.512 | 重みを上げるほど悪化 (-60: 0.332) |
| REACHABLE | -8: 0.463 | 全て baseline 未満 |

→ 単一シードでは ALL の最適が -4 寄りに見えたが (後述の通りシード由来)、REACHABLE/
LOWEST は ALL に劣ることが明白。

### 重み確認: ALL を多シードで (depth5, 1200局/重み)

| weight | -4 | -6 | -8 |
|--------|----|----|----|
| winrate | 0.508 | 0.530 | **0.542** |

→ **多シードでは -8 が最良**。Stage A の「-4 が上」は単一シード (31337) のアーティ
ファクト。「深さで最適重みが小さくなる」仮説は多シードで否定された。再び単一シードに
惑わされない教訓。

### トーナメント (総当たり, depth5, 1600局/設定, opening seeds 11+22)

| 順位 | 設定 | 総合 winrate |
|------|------|-------------|
| 1 | **ALL -8 (既定)** | **0.530** |
| 2 | ALL -6 | 0.528 (統計的に同等) |
| 3 | ALL -10 | 0.515 |
| 4 | base (line_potential) | 0.483 |
| 5 | LOWEST -8 | 0.480 |
| 6 | REACHABLE -8 | 0.463 |

(1SE ≈ 0.013)

### 結論

- **既定 (ALL, weight -8) が最良であることを確認**。ALL -6 は統計的に同等で、-8 据え置きで問題ない。
- **精緻化 (LOWEST / REACHABLE) は行き止まり**: いずれも baseline 未満に沈んだ。パリティ
  の効きは「**潜在的な open-3 脅威の構造全体**」に宿り、最下段や即着手可能な部分集合では
  ない。Connect Four 流の「最下位脅威が支配」直観は **そのままは転移しない**。
- 即時脅威 (immediate) も以前の計測同様に頑健な改善にならず不採用。
- **既定の評価は変更しない** (探索の Rust 既定も Python と同じ ALL/-8)。精緻化の試みは
  負の結果だが、既存選択の頑健性を確認し代替案を排除した点で有益。
- 方法論: 高速化により大標本トーナメントが可能になり、単一シードの誤りを多シード集計で
  確実に検出できるようになった。
