# Threat Quiescence Search の計測 (強化 Phase 2)

> 強化指示書 Phase 2 (B)。depth==0 の地平線で強制手 (即勝ち・唯一の受け・ダブル即勝ち
> 負け・ダブルリーチ生成) だけを `qdepth` まで延長する Threat Quiescence を Python+Rust
> 両方に実装し (`qdepth` 引数、既定 0 = 従来の静的評価)、効果を計測した。

## 実装と契約

- `quiescence` は **窓 (alpha/beta) 非依存の純粋関数**。`negamax` も `negamax_full` も
  同一の葉値を得るため、全幅契約・言語横断契約 (Rust==Python) を維持する。
- 検証: `negamax_value` / `search` が `qdepth∈{0,4,8}` で Rust==Python に一致 (69/69)。
  既定 `qdepth=0` で既存 89 テスト全緑 (挙動・契約は不変)。D4 不変・盤面復元も確認。

## 計測1: 戦術検出率 (浅い探索で強制勝ちを「見える」か)

深さ8で強制勝ちと確認した 50 局面で、浅い探索が勝ちスコア (>MATE_LO) を返す割合。

| depth | qdepth=0 (既定) | qdepth=8 (静穏化) |
|------:|----------------:|------------------:|
| 1 | 0.94 | 0.98 |
| 2 | 0.94 | 0.98 |
| 3 | 0.98 | 0.98 |

→ 静穏化で僅かに向上 (+2局/50)。ただし**既定でも 0.94〜0.98 と既に高い**。既存の脅威枝刈り
(depth>=2 の即勝ち/ダブル即勝ち) と根の即勝ち短絡が、浅い深さでも大半の戦術を捕捉済み。

## 計測2: 固定時間自己対戦 (同一時間での棋力)

A=静穏化(qdepth=8) vs B=既定(qdepth=0)。既定評価。先後入替で対戦。

| 時間 | games | qdepth=8 winrate | 内訳 (A-B, draw) |
|-----:|------:|-----------------:|------------------|
| 50ms | 220 | 0.507 | 111-108, 1 |
| 100ms | 120 | 0.467 | 55-63, 2 |

(1SE ≈ 0.034@220, 0.046@120)

→ **ほぼ中立**。同一時間では静穏化の葉精度向上と、1ノード当たりコスト増 (主探索が浅くなる)
が相殺する。既存の脅威枝刈りが既に戦術を解決しているため、純利得はノイズ水準。

## 判断

- **既定は qdepth=0 (静穏化オフ) のまま**。固定時間で明確な向上が無く (50ms 中立 / 100ms 微減)、
  「測って改善が出なければ採用しない」という方針に従う。負の結果だが、実装は正しく契約も
  維持しており、解析用途やより長い持ち時間での再評価に使える資産として残す。
- 実装 (`quiescence` / `fork_moves` / `qdepth` 引数 / per-side qdepth + 固定時間 `play_match`)
  はそのまま維持。Web/wasm の既定も qdepth=0。
- 残課題: より長い持ち時間 (500〜3000ms) での再評価、静穏化対象の精緻化 (単一脅威生成手の
  選択的延長など) は今後の検討余地。現状は中立のため既定変更はしない。
