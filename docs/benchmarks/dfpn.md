# df-PN 詰み探索ベンチ (Phase 7)

零評価 αβ `solve` の代替として df-PN（depth-first proof-number search）を実装し、
**同じ勝ち判定**を返すことを契約で固めた上で、ノード数・時間を計測した。
CLAUDE.md「改善が確認できない最適化は採用しない／計測で判断」に従う採否記録。

## 実装

`src/score_four/dfpn.py`（純 Python）。明示的 AND/OR の df-PN（Nagai 2002）。

- **明示的 AND/OR（negamax にしない）**: 「手番側が勝つ」は勝ち/それ以外（負け・引分・
  地平線）の二値だが、引分・地平線があると negamax 対称性（相手が勝てない＝自分が勝つ）が
  崩れる。証明者 ME＝根の手番を固定し、OR ノード（ME 手番）は子の1つ、AND ノード（相手
  手番）は全子で証明、とする。
- **Score Four は GHI フリー**: 局面値は (b0,b1) だけで決まり（手番は駒数偶奇で一意）、
  手順に依存しない（重力で一手ごと駒が増える DAG・反復なし）。よって **D4 正規化キーの
  置換表が健全**。地平線 L は根からの絶対手数で測るので転置局面の残り深さも一意。
- **脅威 narrowing を αβ と同形に内蔵**（即勝ち／ダブルリーチ負け／唯一の受け）。

**契約**: `prove_win(b, mp) == (solve(b, mp).status == "win")` を 380 局面（mp 7/9/13・
詰みあり 248／なし 132）で確認、**不一致 0**（`test_dfpn.py`）。

## 計測（`scripts/benchmark_dfpn.py`, max_plies=9）

公平性: 両者とも `Board.play`（局面を1つ生成する操作）の回数を同一カウンタで数える。
αβ は**実デプロイの `solve`**（反復深化・詰みを読み切ったら停止）。df-PN は完全解決まで。

| 局面群 | αβ plays/pos | df-PN plays/pos | df-PN/αβ ノード比 |
|--------|-------------:|----------------:|:-----------------:|
| **詰みあり** setup 14-30 | 7565 | 21 | **0.003** |
| 詰みあり setup 22-38 | 1 | 0 | ~0 |
| **詰みあり** setup 28-44 | 71 | 1 | **0.015** |
| 詰みなし setup 14-30 | 75342 | 312705 | 4.15 |
| 詰みなし setup 22-38 | 17958 | 30132 | 1.68 |
| 詰みなし setup 28-44 | 14923 | 11041 | 0.74 |

## 所見

- **詰みの証明（mate）では df-PN が圧勝**: ノード 0.003〜0.015 倍（最大 ~300 倍少ない）。
  とくに開けた中盤（setup 14-30）で αβ が深く広く探すのに対し df-PN は証明へ直行する。
- **詰みなしの反証（no-mate）では df-PN が劣る**: ノード 1.7〜4.2 倍。地平線内に勝ちが
  無いことの証明（全応手が失敗することの確認）は、αβ の窓カットの方が効率的で、df-PN の
  和ベースの disproof number ＋窓無しが不利になる。
- df-PN は **最短手数・PV を返さない**（証明の有無のみ）。`solve` の API（plies・pv）と
  Web の「詰み手順ハイライト」はこれらに依存する。

## 判断（不採用＝既定は αβ `solve` のまま、df-PN は資産として保持）

df-PN は**詰み発見に限れば桁違いに速い**が、(1) 詰みなしの反証では遅く、(2) 最短手数・PV を
返さないため、`solve` の**ドロップイン置換にはならない**（Web「詰み探索」・問題生成・Rust/
WASM は距離＋PV＋no-mate 効率に依存）。よって **既定は αβ のまま**、df-PN は Phase 2/5/8 と
同様に**検証済みの資産として保持**する。

**今後の活用候補（計測で勝ち越して初めて採用）**:
- 問題生成（`problems.py`）で「強制勝ちか？」のブール判定に df-PN を使い、確定した局面だけ
  αβ で最短手数を求めるハイブリッド（生成の正例検出が大幅高速化しうる）。
- 「詰みを疑う」解析モード（攻めている側の読み）専用の高速オラクル。
- 1+ε トリック・重み付き df-PN・Rust 移植は、上記の採用が決まってから。
